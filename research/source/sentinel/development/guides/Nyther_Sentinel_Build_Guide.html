1. MVP Strategy — What You Are Actually Building

Critical framing

You are NOT building production Nyther Sentinel in 3 months. You are building a compelling proof of the core thesis — enough to show a VC that (1) the technical idea works, (2) you can execute, and (3) the market is real. Every decision in this guide is optimized for that goal.

What the VC demo must show

  1. A simulated power distribution network that behaves like a real Indian grid feeder
  2. Three attack scenarios injected live: FDI attack, electricity theft, malicious SCADA command
  3. AEthelred detecting FDI and theft by catching physics violations in real time
  4. Dvvalin detecting the SCADA command anomaly from network traffic patterns
  5. Valkyrie fusing both signals into a single threat confidence score with an alert
  6. A clean dashboard showing all of this live — not a Jupyter notebook

What you are NOT building for the VC demo

  • Brynhildr (thermodynamic analysis) — mention it as roadmap, build it post-funding
  • Real DISCOM data integration — simulate a realistic Indian feeder topology
  • Production hardening, multi-tenant auth, cloud deployment
  • A mobile app or installer

The core technical decisions

Based on your background (Python/ML, networking, backend — no power systems), here are the deliberate choices made in this guide:

  • Grid simulator: pandapower — Python-native, IEEE test networks built in, no MATLAB needed
  • Digital twin concept: Run the expected state in pandapower, compare against "sensor readings" — the gap is your anomaly score
  • ML approach for AEthelred: Start with physics residuals + Isolation Forest, then upgrade to LSTM autoencoder. This is faster to build and more explainable to VCs
  • Network IDS: Simulate Modbus traffic with pymodbus, detect anomalies with statistical baselines + One-Class SVM
  • No real hardware: Everything runs in software simulation. Frame this correctly — "we simulated a 33-bus Indian distribution feeder with parameters matching CESC topology"

Power systems crash course — what you must understand

You don't need an electrical engineering degree. You need to understand four things deeply enough to build AEthelred:

1. The bus system

A "bus" is a node in the grid — a point where power flows in or out. The IEEE 14-bus test network (your starting point) has 14 buses connected by transmission lines. Each bus has: voltage magnitude (V), voltage angle (θ), active power injection (P), reactive power injection (Q).

2. Power flow equations (the core of AEthelred)

For any bus i in the network, the injected active power must satisfy:

# Active power balance at bus i
# Pi_injected = sum over all connected buses j of:
#   Vi * Vj * (Gij * cos(θi - θj) + Bij * sin(θi - θj))
#
# Where:
#   Vi, Vj = voltage magnitudes at buses i and j
#   θi, θj = voltage angles at buses i and j
#   Gij = conductance of line i-j (real part of admittance)
#   Bij = susceptance of line i-j (imaginary part of admittance)

# In pandapower, this is solved automatically:
import pandapower as pp
net = pp.networks.case14()   # IEEE 14-bus
pp.runpp(net)                # runs Newton-Raphson power flow
print(net.res_bus)           # voltage magnitudes and angles at each bus
print(net.res_line)          # power flows on each line

3. Energy balance (the theft detection primitive)

At any distribution transformer feeding N customers:

# P_in = sum(P_customer_i) + P_line_losses + P_unaccounted
#
# P_line_losses can be estimated: P_loss = I^2 * R
# If P_unaccounted > threshold → theft or false data injection
#
# Key metric: Non-Technical Loss (NTL) ratio
# NTL = (P_in - sum(P_meters) - P_estimated_loss) / P_in
# NTL > 0.05 (5%) is suspicious in Indian distribution networks

4. State estimation vs. true state

The grid has a "true physical state" and an "observed state" from sensors. An FDI attack manipulates sensor readings while the true state stays the same. AEthelred's job is to catch when the observed state is physically impossible given the network topology — the attacker cannot change the laws of physics.

# core/grid_simulator/feeder.py
import pandapower as pp
import pandapower.networks as pn
import numpy as np
import pandas as pd
from dataclasses import dataclass
from typing import Optional

@dataclass
class GridSnapshot:
    timestamp: float
    vm_pu: np.ndarray        # voltage magnitudes per bus
    va_degree: np.ndarray    # voltage angles per bus
    p_mw: np.ndarray         # active power per bus
    q_mvar: np.ndarray       # reactive power per bus
    line_loading: np.ndarray # line loading %
    is_attacked: bool = False
    attack_type: Optional[str] = None

class IndianDistributionFeeder:
    """
    Simulates a 33-bus Indian distribution feeder.
    Uses IEEE 33-bus test case as the topology base,
    with load scaling to match typical Indian urban load profiles.
    """
    def __init__(self):
        self.net = pn.case33bw()  # 33-bus distribution network
        self._scale_to_indian_loads()
        self.baseline_snapshots = []

    def _scale_to_indian_loads(self):
        # Scale loads to reflect typical Indian urban distribution
        # Average household: 1.5-2 kW, 30-100 households per transformer
        self.net.load['p_mw'] *= 0.8   # scale active power
        self.net.load['q_mvar'] *= 0.8  # scale reactive power

    def get_snapshot(self) -> GridSnapshot:
        pp.runpp(self.net, algorithm='bfsw', numba=False)
        return GridSnapshot(
            timestamp=pd.Timestamp.now().timestamp(),
            vm_pu=self.net.res_bus['vm_pu'].values.copy(),
            va_degree=self.net.res_bus['va_degree'].values.copy(),
            p_mw=self.net.res_bus['p_mw'].values.copy(),
            q_mvar=self.net.res_bus['q_mvar'].values.copy(),
            line_loading=self.net.res_line['loading_percent'].values.copy(),
        )

    def simulate_load_cycle(self, hours=24, steps_per_hour=12):
        """Generate baseline data with realistic diurnal load variation."""
        snapshots = []
        for h in range(hours * steps_per_hour):
            t = h / steps_per_hour
            load_factor = self._diurnal_factor(t % 24)
            self.net.load['p_mw'] = self.net.load['p_mw'] * load_factor
            snapshots.append(self.get_snapshot())
            self.net.load['p_mw'] = self.net.load['p_mw'] / load_factor
        self.baseline_snapshots = snapshots
        return snapshots

    def _diurnal_factor(self, hour: float) -> float:
        # Approximate Indian residential load curve
        if 0 <= hour < 6: return 0.5       # night
        elif 6 <= hour < 9: return 0.8      # morning ramp
        elif 9 <= hour < 12: return 0.9     # daytime
        elif 12 <= hour < 15: return 1.1    # afternoon peak (AC load in India)
        elif 15 <= hour < 18: return 0.95
        elif 18 <= hour < 22: return 1.3    # evening peak
        else: return 0.7

class AttackInjector:
    """Injects cyber-physical attacks into the grid simulation."""

    def inject_fdi(self, snapshot: GridSnapshot, target_buses: list,
                   scale_factor: float = 1.2) -> GridSnapshot:
        """
        False Data Injection: inflate voltage readings at target buses.
        This represents an attacker who has compromised sensors at those buses.
        The physical grid is unchanged — only reported measurements are falsified.
        """
        attacked = GridSnapshot(**snapshot.__dict__)
        attacked.vm_pu = snapshot.vm_pu.copy()
        for bus in target_buses:
            attacked.vm_pu[bus] *= scale_factor
        attacked.is_attacked = True
        attacked.attack_type = f"FDI_buses_{target_buses}"
        return attacked

    def inject_electricity_theft(self, snapshot: GridSnapshot,
                                  theft_bus: int, theft_mw: float) -> GridSnapshot:
        """
        Electricity theft: a consumer at theft_bus draws theft_mw of power
        but their smart meter reports zero consumption.
        The feeder-level measurement shows the true power, creating an imbalance.
        """
        attacked = GridSnapshot(**snapshot.__dict__)
        attacked.p_mw = snapshot.p_mw.copy()
        # Feeder input power increases (theft is real), but meter sum doesn't
        attacked.p_mw[0] += theft_mw   # extra power from source
        attacked.is_attacked = True
        attacked.attack_type = f"THEFT_bus_{theft_bus}_{theft_mw}MW"
        return attacked

    def inject_load_manipulation(self, snapshot: GridSnapshot,
                                  buses: list, factor: float) -> GridSnapshot:
        """Simultaneous load manipulation — coordinated attack via compromised meters."""
        attacked = GridSnapshot(**snapshot.__dict__)
        attacked.p_mw = snapshot.p_mw.copy()
        for bus in buses:
            attacked.p_mw[bus] *= factor
        attacked.is_attacked = True
        attacked.attack_type = f"LOAD_MANIP_{buses}"
        return attacked

Week 3 — Physics residual computation + Isolation Forest

# core/aethelred/detector.py
import numpy as np
import pandapower as pp
import pandapower.networks as pn
from sklearn.ensemble import IsolationForest
from sklearn.preprocessing import StandardScaler
from dataclasses import dataclass
from typing import Tuple

@dataclass
class AnomalyResult:
    anomaly_score: float       # 0 = normal, 1 = definite anomaly
    residuals: np.ndarray      # per-bus physics residuals
    violation_buses: list      # buses where KCL/KVL is violated
    energy_imbalance: float    # feeder-level energy imbalance ratio
    detection_type: str        # 'FDI', 'THEFT', 'LOAD_MANIP', 'NORMAL'

class PhysicsResidualEngine:
    """
    Core of AEthelred. Computes how far observed measurements
    deviate from what physics says they should be.
    """
    def __init__(self, net_topology):
        self.net = net_topology
        self.Y_bus = None
        self._build_admittance_matrix()

    def _build_admittance_matrix(self):
        """Build Y-bus admittance matrix for power flow validation."""
        pp.runpp(self.net, numba=False)
        n_buses = len(self.net.bus)
        self.Y_bus = np.zeros((n_buses, n_buses), dtype=complex)
        for _, line in self.net.line.iterrows():
            i = int(line['from_bus'])
            j = int(line['to_bus'])
            z = complex(line['r_ohm_per_km'], line['x_ohm_per_km']) * line['length_km'] / 1000
            if abs(z) < 1e-10:
                continue
            y = 1 / z
            self.Y_bus[i][i] += y
            self.Y_bus[j][j] += y
            self.Y_bus[i][j] -= y
            self.Y_bus[j][i] -= y

    def compute_power_residuals(self, observed_vm: np.ndarray,
                                 observed_va_deg: np.ndarray,
                                 reported_p_mw: np.ndarray) -> np.ndarray:
        """
        Given observed voltages, compute expected power injections using Y_bus.
        Compare against reported power injections.
        Residual = |P_expected - P_reported| at each bus.
        """
        va_rad = np.deg2rad(observed_va_deg)
        V = observed_vm * np.exp(1j * va_rad)
        S_calc = V * np.conj(self.Y_bus @ V)  # S = V * (Y*V)*
        P_calc = S_calc.real  # active power (MW) per bus

        P_reported = reported_p_mw / self.net.sn_mva  # normalize to per-unit
        residuals = np.abs(P_calc - P_reported)
        return residuals

    def compute_energy_imbalance(self, p_in_mw: float,
                                  meter_readings_mw: np.ndarray,
                                  estimated_loss_mw: float) -> float:
        """
        Feeder-level energy balance check.
        Returns NTL ratio — > 0.05 is suspicious.
        """
        p_accounted = np.sum(meter_readings_mw) + estimated_loss_mw
        imbalance = (p_in_mw - p_accounted) / (p_in_mw + 1e-6)
        return imbalance

    def check_kirchhoff_violations(self, snapshot) -> list:
        """
        Kirchhoff's Current Law: current into node = current out of node.
        Find buses where this is violated beyond threshold.
        """
        violations = []
        threshold = 0.05  # 5% tolerance
        for bus_idx in range(len(self.net.bus)):
            # Sum of currents at each bus should be near zero
            va_rad = np.deg2rad(snapshot.va_degree)
            V = snapshot.vm_pu * np.exp(1j * va_rad)
            I_total = self.Y_bus[bus_idx] @ V
            if abs(I_total.real) > threshold:
                violations.append(bus_idx)
        return violations


class AEthelredDetector:
    """
    Full AEthelred detection pipeline.
    Combines physics residuals with ML-based anomaly detection.
    """
    def __init__(self, net_topology):
        self.physics_engine = PhysicsResidualEngine(net_topology)
        self.scaler = StandardScaler()
        self.isolation_forest = IsolationForest(
            n_estimators=200,
            contamination=0.05,  # expect ~5% anomalous readings
            random_state=42
        )
        self.is_fitted = False
        self.thresholds = {
            'residual_mean': 0.03,     # 3% mean residual → suspicious
            'residual_max': 0.08,      # 8% max residual at any bus → alert
            'energy_imbalance': 0.05,  # 5% energy imbalance → alert
        }

    def fit(self, normal_snapshots: list):
        """Train the detector on baseline normal operation data."""
        features = [self._extract_features(s) for s in normal_snapshots]
        X = np.array(features)
        X_scaled = self.scaler.fit_transform(X)
        self.isolation_forest.fit(X_scaled)
        self.is_fitted = True
        print(f"AEthelred trained on {len(normal_snapshots)} normal snapshots")

    def _extract_features(self, snapshot) -> np.ndarray:
        """Extract feature vector from a grid snapshot."""
        residuals = self.physics_engine.compute_power_residuals(
            snapshot.vm_pu, snapshot.va_degree, snapshot.p_mw
        )
        features = np.concatenate([
            snapshot.vm_pu,           # voltage magnitudes
            np.abs(snapshot.p_mw),    # active power
            residuals,                 # physics residuals
            [snapshot.vm_pu.std(),     # global voltage deviation
             np.max(residuals),        # worst-case residual
             np.mean(residuals)]       # average residual
        ])
        return features

    def detect(self, snapshot) -> AnomalyResult:
        """Run detection on a new snapshot. Returns AnomalyResult."""
        residuals = self.physics_engine.compute_power_residuals(
            snapshot.vm_pu, snapshot.va_degree, snapshot.p_mw
        )
        energy_imbalance = self.physics_engine.compute_energy_imbalance(
            p_in_mw=abs(snapshot.p_mw[0]),
            meter_readings_mw=np.abs(snapshot.p_mw[1:]),
            estimated_loss_mw=0.02 * abs(snapshot.p_mw[0])
        )
        kcl_violations = self.physics_engine.check_kirchhoff_violations(snapshot)

        # Physics-based score (deterministic)
        physics_score = 0.0
        if np.mean(residuals) > self.thresholds['residual_mean']:
            physics_score += 0.4
        if np.max(residuals) > self.thresholds['residual_max']:
            physics_score += 0.3
        if abs(energy_imbalance) > self.thresholds['energy_imbalance']:
            physics_score += 0.3

        # ML-based score (learned from data)
        ml_score = 0.0
        if self.is_fitted:
            features = self._extract_features(snapshot).reshape(1, -1)
            X_scaled = self.scaler.transform(features)
            raw_score = self.isolation_forest.decision_function(X_scaled)[0]
            ml_score = max(0, min(1, -raw_score + 0.5))

        # Combine
        final_score = 0.6 * physics_score + 0.4 * ml_score

        # Classify
        detection_type = 'NORMAL'
        if energy_imbalance > 0.07:
            detection_type = 'THEFT'
        elif np.max(residuals) > self.thresholds['residual_max']:
            detection_type = 'FDI'
        elif physics_score > 0.5:
            detection_type = 'ANOMALY'

        return AnomalyResult(
            anomaly_score=final_score,
            residuals=residuals,
            violation_buses=kcl_violations,
            energy_imbalance=energy_imbalance,
            detection_type=detection_type
        )

Week 4 — Upgrade to LSTM autoencoder for temporal patterns

Isolation Forest works on single snapshots. An LSTM autoencoder learns normal time-series patterns and flags deviations. This is significantly more powerful and more impressive in a demo.

# core/aethelred/lstm_autoencoder.py
import torch
import torch.nn as nn
import numpy as np

class LSTMAutoencoder(nn.Module):
    """
    LSTM autoencoder for time-series anomaly detection.
    Trained on normal grid operation windows (e.g., 60-second windows).
    High reconstruction error = anomaly.
    """
    def __init__(self, input_dim: int, hidden_dim: int = 64,
                 num_layers: int = 2, sequence_len: int = 30):
        super().__init__()
        self.sequence_len = sequence_len
        self.hidden_dim = hidden_dim

        # Encoder
        self.encoder = nn.LSTM(
            input_size=input_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            dropout=0.2
        )
        # Decoder
        self.decoder = nn.LSTM(
            input_size=hidden_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            dropout=0.2
        )
        self.output_layer = nn.Linear(hidden_dim, input_dim)

    def forward(self, x):
        # x shape: (batch, seq_len, input_dim)
        encoded, (h_n, c_n) = self.encoder(x)
        # Use encoder's last hidden state to seed decoder
        decoder_input = encoded[:, -1:, :].repeat(1, self.sequence_len, 1)
        decoded, _ = self.decoder(decoder_input, (h_n, c_n))
        reconstruction = self.output_layer(decoded)
        return reconstruction

    def reconstruction_error(self, x: torch.Tensor) -> torch.Tensor:
        """Per-sample mean squared reconstruction error."""
        recon = self.forward(x)
        return torch.mean((x - recon) ** 2, dim=[1, 2])


def train_lstm_autoencoder(model, normal_sequences, epochs=50, lr=1e-3):
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    loss_fn = nn.MSELoss()
    model.train()
    for epoch in range(epochs):
        total_loss = 0
        for seq in normal_sequences:
            x = torch.FloatTensor(seq).unsqueeze(0)
            optimizer.zero_grad()
            recon = model(x)
            loss = loss_fn(recon, x)
            loss.backward()
            optimizer.step()
            total_loss += loss.item()
        if epoch % 10 == 0:
            print(f"Epoch {epoch}: loss = {total_loss / len(normal_sequences):.6f}")
    return model

@dataclass
class ModbusPacket:
    timestamp: float
    src_ip: str
    dst_ip: str
    function_code: int      # e.g., 3 = Read Holding Registers, 16 = Write Multiple
    unit_id: int            # device address
    register_address: int
    register_count: int
    data: bytes
    is_malicious: bool = False
    attack_type: str = None

NORMAL_FUNCTION_CODES = {
    1: "Read Coils",
    2: "Read Discrete Inputs",
    3: "Read Holding Registers",   # most common in SCADA monitoring
    4: "Read Input Registers",
    5: "Write Single Coil",        # circuit breaker control
    6: "Write Single Register",
}

SUSPICIOUS_PATTERNS = {
    # Unexpected write commands to critical registers
    'unauthorized_write': lambda pkt: pkt.function_code in [5, 6, 15, 16]
                                      and pkt.unit_id in [1, 2, 3],  # critical PLCs
    # Scanning behavior (rapid sequential reads across many registers)
    'register_scan': None,  # detected by time-series analysis
    # Unusual function codes (could be exploits)
    'rare_function_code': lambda pkt: pkt.function_code not in NORMAL_FUNCTION_CODES,
}

class TrafficProfiler:
    """
    Learns the baseline communication profile for each device pair.
    Detects deviations: timing, frequency, function codes, register ranges.
    """
    def __init__(self):
        self.profiles = {}   # key: (src_ip, dst_ip, unit_id)

    def update_profile(self, packet: ModbusPacket):
        key = (packet.src_ip, packet.dst_ip, packet.unit_id)
        if key not in self.profiles:
            self.profiles[key] = {
                'function_codes': {},
                'register_ranges': [],
                'inter_arrival_times': [],
                'last_seen': packet.timestamp,
            }
        p = self.profiles[key]
        fc = packet.function_code
        p['function_codes'][fc] = p['function_codes'].get(fc, 0) + 1
        p['register_ranges'].append((packet.register_address,
                                     packet.register_address + packet.register_count))
        if p['last_seen']:
            iat = packet.timestamp - p['last_seen']
            p['inter_arrival_times'].append(iat)
        p['last_seen'] = packet.timestamp

    def get_expected_profile(self, key):
        if key not in self.profiles:
            return None
        p = self.profiles[key]
        total = sum(p['function_codes'].values())
        fc_distribution = {k: v/total for k, v in p['function_codes'].items()}
        iats = p['inter_arrival_times']
        return {
            'fc_distribution': fc_distribution,
            'mean_iat': sum(iats)/len(iats) if iats else None,
            'std_iat': (sum((x-sum(iats)/len(iats))**2 for x in iats)/len(iats))**0.5 if len(iats) > 1 else 0,
            'typical_registers': p['register_ranges'],
        }

from sklearn.preprocessing import StandardScaler
from collections import deque
from dataclasses import dataclass
from typing import List, Optional

@dataclass
class NetworkAnomalyResult:
    anomaly_score: float
    triggered_rules: List[str]
    anomaly_type: Optional[str]
    confidence: float

class DvvalinDetector:
    """
    Multi-layer SCADA network intrusion detection.
    Layer 1: Rule-based (fast, deterministic, handles known attack patterns)
    Layer 2: Statistical baseline (timing and frequency anomalies)
    Layer 3: One-Class SVM (learned behavioral model)
    """
    def __init__(self):
        self.profiler = TrafficProfiler()
        self.scaler = StandardScaler()
        self.ocsvm = OneClassSVM(kernel='rbf', gamma='auto', nu=0.05)
        self.packet_window = deque(maxlen=100)  # sliding window
        self.is_fitted = False
        self.rules = self._define_rules()

    def _define_rules(self):
        return [
            {
                'name': 'unauthorized_write_to_critical_device',
                'condition': lambda p: p.function_code in [5, 6, 15, 16]
                                       and p.unit_id <= 3,
                'score': 0.9,
                'type': 'SCADA_COMMAND_INJECTION'
            },
            {
                'name': 'unknown_function_code',
                'condition': lambda p: p.function_code not in [1,2,3,4,5,6,15,16,43],
                'score': 0.7,
                'type': 'PROTOCOL_ANOMALY'
            },
            {
                'name': 'device_impersonation',
                'condition': lambda p: self._check_new_device(p),
                'score': 0.8,
                'type': 'DEVICE_SPOOFING'
            },
        ]

    def _check_new_device(self, packet: ModbusPacket) -> bool:
        """Flag packets from device IDs never seen before in training."""
        key = (packet.src_ip, packet.unit_id)
        return key not in self.known_devices if hasattr(self, 'known_devices') else False

    def _extract_features(self, packet: ModbusPacket,
                           window: deque) -> np.ndarray:
        """Extract feature vector from a packet + recent context window."""
        recent = list(window)
        fc_freq = np.bincount([p.function_code for p in recent] + [0],
                               minlength=65)[:20] / max(len(recent), 1)
        unit_freq = np.zeros(20)
        for p in recent:
            if p.unit_id < 20:
                unit_freq[p.unit_id] += 1
        unit_freq /= max(len(recent), 1)
        iats = np.diff([p.timestamp for p in recent]) if len(recent) > 1 else [0.1]
        features = np.concatenate([
            fc_freq,
            unit_freq,
            [np.mean(iats), np.std(iats),
             packet.function_code / 65.0,
             packet.unit_id / 256.0,
             packet.register_address / 65535.0,
             packet.register_count / 125.0]
        ])
        return features

    def fit(self, normal_packets: List[ModbusPacket]):
        """Train on normal SCADA traffic patterns."""
        self.known_devices = set((p.src_ip, p.unit_id) for p in normal_packets)
        for packet in normal_packets:
            self.profiler.update_profile(packet)
        window = deque(maxlen=100)
        features = []
        for packet in normal_packets:
            window.append(packet)
            features.append(self._extract_features(packet, window))
        X = np.array(features)
        X_scaled = self.scaler.fit_transform(X)
        self.ocsvm.fit(X_scaled)
        self.is_fitted = True

    def detect(self, packet: ModbusPacket) -> NetworkAnomalyResult:
        self.packet_window.append(packet)
        triggered_rules = []
        rule_score = 0.0
        for rule in self.rules:
            try:
                if rule['condition'](packet):
                    triggered_rules.append(rule['name'])
                    rule_score = max(rule_score, rule['score'])
            except Exception:
                pass

        ml_score = 0.0
        if self.is_fitted:
            features = self._extract_features(packet, self.packet_window).reshape(1, -1)
            X_scaled = self.scaler.transform(features)
            raw = self.ocsvm.decision_function(X_scaled)[0]
            ml_score = max(0, min(1, -raw + 0.5))

        final_score = 0.7 * rule_score + 0.3 * ml_score if triggered_rules else ml_score
        anomaly_type = triggered_rules[0].upper() if triggered_rules else (
            'BEHAVIORAL_ANOMALY' if ml_score > 0.6 else None
        )
        return NetworkAnomalyResult(
            anomaly_score=final_score,
            triggered_rules=triggered_rules,
            anomaly_type=anomaly_type,
            confidence=0.9 if triggered_rules else 0.6
        )

Key metrics to compute and report:
- Detection rate (recall): % of attacks caught
- False positive rate: % of normal traffic flagged
- F1 score
- Mean time to detect (MTTD): seconds from attack start to first alert

import pyshark
from sklearn.metrics import classification_report, confusion_matrix

def parse_pcap_to_modbus_packets(pcap_file: str, label: bool) -> List[ModbusPacket]:
    packets = []
    cap = pyshark.FileCapture(pcap_file, display_filter='modbus')
    for pkt in cap:
        try:
            mp = ModbusPacket(
                timestamp=float(pkt.sniff_timestamp),
                src_ip=str(pkt.ip.src),
                dst_ip=str(pkt.ip.dst),
                function_code=int(pkt.modbus.func_code),
                unit_id=int(pkt.modbus.unit_id),
                register_address=int(getattr(pkt.modbus, 'regaddr', 0)),
                register_count=int(getattr(pkt.modbus, 'wordcnt', 1)),
                data=b'',
                is_malicious=label
            )
            packets.append(mp)
        except AttributeError:
            continue
    return packets

# core/valkyrie/fusion.py
import numpy as np
from dataclasses import dataclass, field
from typing import List, Dict, Optional
from enum import Enum
import time

class ThreatLevel(Enum):
    NORMAL = "NORMAL"
    LOW = "LOW"
    MEDIUM = "MEDIUM"
    HIGH = "HIGH"
    CRITICAL = "CRITICAL"

@dataclass
class ThreatEvent:
    timestamp: float
    threat_level: ThreatLevel
    final_score: float
    aethelred_score: float
    dvvalin_score: float
    attack_type: str
    affected_buses: List[int]
    recommended_actions: List[str]
    confidence: float

class ValkyrieFusion:
    """
    Multi-domain fusion engine.
    Combines physics, network, and (future) thermodynamic signals.

    Key design: weights are adaptive based on confidence levels.
    If Dvvalin has high confidence (rule triggered), its weight increases.
    If only statistical anomaly (ML only), weight is lower.
    """
    def __init__(self):
        # Base weights — can be tuned per deployment
        self.base_weights = {
            'aethelred': 0.55,   # physics is primary signal
            'dvvalin': 0.45,     # network is corroborating
            # 'brynhildr': 0.20  # thermodynamic — future
        }
        self.alert_history = []
        self.thresholds = {
            ThreatLevel.LOW: 0.25,
            ThreatLevel.MEDIUM: 0.45,
            ThreatLevel.HIGH: 0.65,
            ThreatLevel.CRITICAL: 0.80,
        }
        self.cooldown_period = 30  # seconds between alerts for same type
        self._last_alerts: Dict[str, float] = {}

    def fuse(self, aethelred_result, dvvalin_result,
             aethelred_confidence: float = 0.7,
             dvvalin_confidence: float = 0.7) -> ThreatEvent:

        # Adaptive weighting based on confidence
        w_a = self.base_weights['aethelred'] * aethelred_confidence
        w_d = self.base_weights['dvvalin'] * dvvalin_confidence
        total = w_a + w_d
        w_a /= total
        w_d /= total

        final_score = w_a * aethelred_result.anomaly_score + \
                      w_d * dvvalin_result.anomaly_score

        # Corroboration bonus: if both signals agree, amplify confidence
        if (aethelred_result.anomaly_score > 0.5 and
                dvvalin_result.anomaly_score > 0.5):
            final_score = min(1.0, final_score * 1.25)

        threat_level = ThreatLevel.NORMAL
        for level in [ThreatLevel.CRITICAL, ThreatLevel.HIGH,
                      ThreatLevel.MEDIUM, ThreatLevel.LOW]:
            if final_score >= self.thresholds[level]:
                threat_level = level
                break

        attack_type = self._determine_attack_type(aethelred_result, dvvalin_result)
        actions = self._recommend_actions(threat_level, attack_type, aethelred_result)
        confidence = (aethelred_confidence + dvvalin_confidence) / 2 * \
                     (1.3 if final_score > 0.5 and
                      aethelred_result.anomaly_score > 0.3 and
                      dvvalin_result.anomaly_score > 0.3 else 1.0)

        event = ThreatEvent(
            timestamp=time.time(),
            threat_level=threat_level,
            final_score=final_score,
            aethelred_score=aethelred_result.anomaly_score,
            dvvalin_score=dvvalin_result.anomaly_score,
            attack_type=attack_type,
            affected_buses=aethelred_result.violation_buses,
            recommended_actions=actions,
            confidence=min(0.99, confidence)
        )
        self.alert_history.append(event)
        return event

    def _determine_attack_type(self, ae, dv) -> str:
        if dv.anomaly_type == 'SCADA_COMMAND_INJECTION' and ae.detection_type == 'FDI':
            return 'COORDINATED_CYBER_PHYSICAL_ATTACK'
        elif ae.detection_type == 'THEFT':
            return 'ELECTRICITY_THEFT'
        elif ae.detection_type == 'FDI':
            return 'FALSE_DATA_INJECTION'
        elif dv.anomaly_type:
            return dv.anomaly_type
        return 'UNKNOWN_ANOMALY'

    def _recommend_actions(self, level: ThreatLevel,
                            attack_type: str,
                            ae_result) -> List[str]:
        actions = []
        if level in [ThreatLevel.HIGH, ThreatLevel.CRITICAL]:
            actions.append("ISOLATE_AFFECTED_BUSES")
            actions.append("ALERT_GRID_OPERATOR")
            actions.append("CAPTURE_NETWORK_FORENSICS")
        if attack_type == 'ELECTRICITY_THEFT':
            buses_str = ', '.join(map(str, ae_result.violation_buses))
            actions.append(f"FLAG_METERS_AT_BUSES: {buses_str}")
            actions.append("DISPATCH_PHYSICAL_INSPECTION")
        if attack_type == 'FALSE_DATA_INJECTION':
            actions.append("SWITCH_TO_BACKUP_SENSORS")
            actions.append("VERIFY_STATE_ESTIMATION_INTEGRITY")
        if attack_type == 'COORDINATED_CYBER_PHYSICAL_ATTACK':
            actions.append("ESCALATE_TO_NCIIPC")
            actions.append("ACTIVATE_INCIDENT_RESPONSE_PLAYBOOK")
        return actions

10. Common Pitfalls

Pitfall 1: pandapower convergence failures

Power flow may not converge if load values are extreme. Always add max_iteration=50 and catch pp.LoadflowNotConverged. Use the 'bfsw' (backward/forward sweep) algorithm for distribution networks — it's more stable than Newton-Raphson for radial networks like the 33-bus.

Pitfall 2: False positive explosion from load transients

Your AEthelred will trigger constantly at morning/evening load ramps if thresholds are too tight. Add a 30-second rolling average smoother before computing residuals. The LSTM autoencoder handles this naturally — it learns the diurnal cycle.

Pitfall 3: Modbus simulation not realistic enough

Real SCADA traffic has device-specific polling intervals (RTU 1 polled every 500ms, RTU 2 every 2000ms). If your simulation uses uniform timing, Dvvalin will be over-optimistic. Build per-device timing profiles from the MSU dataset's inter-arrival times.

Pitfall 4: Selling it wrong to VCs

Don't say "we built a SCADA IDS." Say "we built the only cyber-physical security platform that validates electrical physics in real time — the first system that makes it impossible to simultaneously manipulate both the cyber and physical layers of a power grid without detection." The differentiation must be crisp in the first 30 seconds.

Pitfall 5: Jupyter notebook as demo

Never demo from a Jupyter notebook in front of a VC. It signals "research project," not "product." The dashboard must look like software someone would pay for. Invest the Week 10-12 time in the UI — it matters more than an extra percentage point of accuracy for the VC pitch.

Pro tip: Record a backup demo video

On Day 80, record a clean 8-minute run-through of the entire demo working perfectly. If anything crashes during a real VC meeting, play the video. "We had a technical issue but here's the recorded demo from yesterday" is infinitely better than a broken live demo.
